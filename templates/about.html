Description

The subway is integral to New York City, shuttling over 3 million people every day. However, it isn't fully equitable. Often, the communities most in need have the least access to this vital public service.
This project analyzes the New York City subway system, identifying inequities and imagining a more equitable and efficient subway system. The "New York City Subway" refers to the rail transit system connecting NYC and operating fully inside NYC (excluding commuter rail networks such as the Long Island Rail Road), and a line on the subway is a path followed by trains, such as the 1, 2, 3, A, B, C, etc.

Prototyping

The project began with a lo-fi prototype of the micro pages (borough data) and the macro page (citywide data). To make the numbers more meaningful, information was overlayed on a map which features subway lines, so discrepancies in connectivity and usage could be emphasized. Additonally, it made it easier to glean information from the map as opposed to having to sift through charts.

Refining

The above prototype was later revised to allow for a clearer and more informative glimpse into the subway system, where each micro page relfected a borough rather than a station. This was due to some feedback, which questioned whether a micro page for each station was informative or useful

Data Cleansing Pipeline

Data used was published through the MTA's Open Data Program. In particular, the "MTA Subway Hourly Ridership: Beginning February 2022" was used. As the dataset was very large, python's requests library was used to communicate with the API and obtain data for one day: March 1st, 2023 (python file). The data was saved as a CSV. The CSV was then opened using pandas, reformatted so it would be easier to work with and saved in JSON format (python file). In particular, the resulting JSON object contains nested dictionaries of station and time.
