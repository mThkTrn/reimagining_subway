Improved Subway Map

{% include "new_map.html" %}

The above map features a more equitable design for the New York City Subway. Some simplifying assumptions were made (only 5 lines, and lines don't intersect). Every dot above corresponds to a location a line passes through. The map has obvious limitations, given the limited nature of the data, the non-intersecting lines and impracticality of its implementation, but it serves as a beginning for discussions on how we can improve out current subway system. The map was algorithmically generated using this script, using the following procedure:

  1. The script defined a utility function that considered the value of adding a new line between any two stations, doing so by analyzing the time profiles of each station.
  2. The script was instructed to keep the number of distinct lines in the current NYC subway (11)
  3. The 11 stations with the least sub of pairwise utilities were identified as the beginning points of the 11 different lines
  4. A greedy algorithm was employed, extending the lines until they encompass all stations in the New York Subway

Unlike the New York City subway, the above map features lines that heavily go through underserved areas, such as the red line which passes through the majority of Brooklyn, Queens and Harlem. This corroborates with the data above, which suggests that more lines should pass through those areas.



Takeaways

  1. The design of the New York City subway is not designed to meet the needs of low income riders
  2. To make it more equitable, attention should be paid to the average time riders board a train in each borough, as it reveals the role that borough plays in people's daily commutes
  3. In particular, more lines should be contructed to connect stations in outer boroughs, as to lessen the strain on stations with high bus-to-train transfers
